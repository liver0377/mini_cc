# 任务队列 (task)

任务队列服务提供持久化的任务管理，支持状态机流转、依赖追踪和循环检测。任务状态通过 JSON 文件持久化，使用文件锁确保并发安全。

## 模块结构

```
task/
├── __init__.py      # 公共导出
└── service.py       # TaskService 实现
```

## 任务状态机

```
                    ┌─────────┐
                    │ pending │
                    └────┬────┘
                         │ start()
                         ▼
                    ┌─────────┐
            ┌──────│ running │──────┐
            │      └─────────┘      │
            │                       │
     complete()               fail() / cancel()
            │                       │
            ▼                       ▼
     ┌───────────┐          ┌───────────┐
     │ completed │          │  failed   │
     └───────────┘          └───────────┘
                                  ▲
                                  │
                           cancel()
                                  │
                                  ▼
                           ┌───────────┐
                           │ cancelled │
                           └───────────┘
```

**合法的状态转换：**

| 起始状态 | 目标状态 | 方法 | 说明 |
|----------|----------|------|------|
| pending | running | `start()` | 开始执行任务 |
| running | completed | `complete()` | 任务成功完成 |
| running | failed | `fail()` | 任务执行失败 |
| pending | cancelled | `cancel()` | 取消待执行任务 |
| running | cancelled | `cancel()` | 取消执行中任务 |

## 依赖追踪

任务可声明对其他任务的依赖，系统确保依赖任务完成后才开始执行：

```
任务依赖关系示例：

  t1 (无依赖) ──────────► t3 (依赖 t1, t2)
                            │
  t2 (无依赖) ──────────────┘
                            │
                            ▼
                         t4 (依赖 t3)

执行顺序：t1, t2 → t3 → t4
```

**依赖管理功能：**

| 功能 | 说明 |
|------|------|
| 依赖声明 | 创建任务时指定 `dependencies` 列表 |
| 就绪检查 | 所有依赖任务状态为 `completed` 时，当前任务才可执行 |
| 循环检测 | 添加依赖时检测是否有环，防止死锁 |
| 依赖查询 | 查询某任务的所有前置依赖和后继任务 |

## 并发安全

```
TaskService 写操作流程：

┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  获取文件锁   │────►│ 读取 JSON    │────►│ 修改内存数据  │
│  fcntl.flock │     │              │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
                                                │
┌──────────────┐     ┌──────────────┐     ┌──────▼───────┐
│  释放文件锁   │◄────│ 写入 JSON    │◄────│ 验证状态转换  │
│              │     │              │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
```

**安全措施：**

| 措施 | 说明 |
|------|------|
| 文件锁 | `fcntl.flock` 保证同一时间只有一个进程写入 |
| 原子读写 | 读取 → 修改 → 写入的完整操作在锁内完成 |
| 状态验证 | 状态转换前验证合法性 |
| 循环检测 | 添加依赖时进行 DAG 验证 |

## TaskService API

| 方法 | 说明 |
|------|------|
| `create_task()` | 创建新任务（含类型、描述、依赖） |
| `start()` | 将任务状态从 pending 转为 running |
| `complete()` | 将任务标记为完成 |
| `fail()` | 将任务标记为失败 |
| `cancel()` | 取消任务 |
| `get_task()` | 获取单个任务详情 |
| `list_tasks()` | 列出所有任务 |
| `get_ready_tasks()` | 获取所有依赖已满足的就绪任务 |
| `add_dependency()` | 添加任务依赖关系（含循环检测） |
